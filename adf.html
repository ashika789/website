Adaptive Design

iOS library that enables web application hosted inside Mobile Container to bring in the flexibility of iOS platform/native specific features, look & feel and transitions

Capabilities

Action Sheet

This iOS native control allows showing an action sheet included in the cocoa framework.

Chart Manager

This iOS native control provides a native view that allows custom charts to be displayed. The chart types supported at the moment are line, bar, circle and pie charts.

Collection View

    This iOS native control provides a native view that allows manage collections of ordered data. This data can be presented to the user in two ways using a grid or a coverFlow.

Expandable Select Button

This iOS native control allows showing a button like UIButton included in the cocoa framework that expands a list of option buttons on tap and hold.

File Reader

This control provides iOS native view that allows displaying a given file. Currently it supports,
  • iWork documents
  • Microsoft Office documents (Office ‘97 and newer)
  • Rich Text Format (RTF) documents
  • PDF files
  • Images
  • Text files whose uniform type identifier (UTI) conforms to the public.text type (see Uniform Type Identifiers Reference)
  • Comma-separated value (csv) files

Form

This iOS native control allows displaying a native form. It could contain any component and a submit button. The data of all the input components is posted in a notification when the submit button is tapped

Image View

This iOS native control allows displaying an image. It could be from a local path or a remote one (in this case the framework sets a placeholder image until the image is fully loaded, and if the download fails, a "content unavailable" image is displayed instead).

Map View

This iOS native control allows showing the native map in the screen. The component is capable of showing the user location in the map and also adds custom annotations using the JSON configuration. The component uses Mapkit so it allows the pinch to zoom action. This control requires an internet connection to load the map.

Media Player

This iOS native control provides a native view that allows media player to be displayed. This component offers reproduce a media resource such as mp3, mp4 and so on.

Modal

This iOS native control allows the user to open any existing control inside a full screen modal control.

PDF Reader

This iOS native control allows opening a pdf document using PDF reader natively available. The control allows pinch to zoom actions and the user can jump to any page using the bottom bar of the reader where icons for all the pages are shown.

Scroll View

This iOS native control provides a parent view that allows containing other components. The view can be scrollable and paginatable. All of this is configurable by JSON.

Slider

This iOS native control allows showing a slider bar, included in the cocoa framework. Also, it allows to add a title and the label with the selected value. It is configured using a JSON file.

Split Drag Container

This iOS native control allows showing two views with a set of components and allows dragging these components between the views.

Tab Bar

This iOS native control provides a native view that allows presenting multiple switchable views accesible from the buttons bar.

Table View

This iOS native control provides a native view that allows custom tables to be displayed. This component offers a table with customizable cells which load the given data input.

Text Field

This iOS native control allows showing a textbox like the "UITextField", included in the cocoa framework. It is configured using a JSON file.

Toggle

This iOS native control allows showing a toggle button, included in the cocoa framework.

Transition Button

This iOS native control that creates a button that can open another control in a modal using a custom transition when the button is pressed.

Button

This iOS native control allows showing a button like UIButton included in the cocoa framework.

Label

This component allows showing a label like UILabel included in the cocoa framework but configured using a JSON file.
